# Testing Design — super-transcribe

**Date:** 2026-02-22
**Approach:** Option A — Layered pytest suite (unit + model-gated integration)

## Goals

Comprehensively test the super-transcribe skill before public release. Cover all shared lib
logic with fast modelless unit tests, and validate the full transcription pipeline end-to-end
with integration tests that auto-skip when ML models are not installed.

## Structure

```
tests/
├── conftest.py                  # shared fixtures: synthetic WAV, sample segments, mock args
├── fixtures/
│   └── tone.wav                 # 1-sec 440Hz sine wave, 16kHz mono — generated by conftest.py
├── unit/
│   ├── test_formatters.py       # all 10 output formats + timestamp helpers + agent JSON
│   ├── test_postprocess.py      # hallucination filter, filler, paragraphs, merge, chapters, search
│   ├── test_audio.py            # probe_audio, is_url, resolve_inputs, convert_to_wav, preprocess
│   ├── test_speakers.py         # apply_speaker_names, export_speakers_audio
│   ├── test_rss.py              # fetch_rss_episodes (mocked HTTP)
│   └── test_router.py           # bash router routing logic (subprocess + stub backends)
└── integration/
    ├── conftest.py              # model-availability checks, skip markers
    ├── test_fw_e2e.py           # faster-whisper end-to-end on tone.wav
    └── test_pk_e2e.py           # parakeet end-to-end on tone.wav

pytest.ini                       # markers: unit, integration; default runs unit only
requirements-test.txt            # pytest, pytest-mock, numpy
```

## Tooling

- **pytest** with **pytest-mock** for mocking
- **numpy** + stdlib `wave` to generate the synthetic WAV fixture (no extra audio deps)
- `pytest.ini` sets `addopts = -m "not integration"` — `pytest` runs unit only by default
- `pytest -m integration` runs the full integration layer (requires models installed)

## Unit Test Coverage

### test_formatters.py
- Timestamp formatters (`format_ts_srt`, `format_ts_vtt`, `format_ts_ass`, `format_ts_ttml`):
  zero-padding, milliseconds vs centiseconds, edge cases (0s, 3600s, fractional seconds)
- `to_srt/vtt/ass/ttml`: with/without speaker labels, `max_words_per_line`,
  `max_chars_per_line`, empty segments list
- `to_text`: no speakers, with speakers, with paragraph breaks, speaker transitions
- `to_csv/tsv`: header correctness, speaker column present/absent, timestamp precision
- `to_lrc`: `[MM:SS.cs]` format, speaker labels
- `to_html`: confidence color classes (`conf-high/med/low`), word-level vs segment-level,
  XSS-safe content
- `to_json`: valid JSON, roundtrippable
- `format_agent_json`: all required fields, `summary_hint` only when >400 chars,
  `avg_confidence` from `avg_logprob` and word probabilities, `file_path`/`output_path` echoed
- `split_words_by_chars`: chunk boundaries, single word exceeding limit

### test_postprocess.py
- `filter_hallucinations`: each HALLUCINATION_PATTERNS entry matches, consecutive duplicates
  removed, clean segments pass through
- `remove_filler_words`: `um/uh/er/ah/hmm`, bigrams `you know/I mean/you see`,
  punctuation cleanup after removal, word list filtering
- `detect_paragraphs`: gap >= min_gap triggers new paragraph, sentence-ending + short gap,
  first segment always gets `paragraph_start=True`
- `merge_sentences`: merges across MAX_GAP=2.0s, flushes on terminal punctuation,
  speaker majority voting in flush
- `detect_chapters`: gap threshold, single chapter when no sufficient gaps, multiple chapters,
  chapter timestamps
- `search_transcript`: exact match, case-insensitive, no match, fuzzy match (ratio >= 0.6),
  multi-word query fuzzy

### test_audio.py (subprocesses mocked)
- `is_url`: http/https/www prefix → True; file paths → False
- `resolve_inputs`: glob expansion, directory traversal (only audio exts), non-audio file
  warning, URL passthrough, missing file warning
- `probe_audio`: mocked ffprobe JSON output → correct field mapping; fallback when ffprobe
  absent; `duration_human` formatting (seconds/minutes/hours)
- `convert_to_wav`: native ext (`.wav`, `.flac`) → passthrough; convertible ext → ffmpeg
  called with correct args; ffmpeg failure → returns original path
- `preprocess_audio`: no-op when no flags; correct filter string for denoise/normalize/both;
  ffmpeg failure fallback
- `extract_channel`: `mix` → passthrough; `left`/`right` → correct `pan=mono|c0=c0/c1` filter

### test_speakers.py
- `apply_speaker_names`: SPEAKER_1→Alice, SPEAKER_2→Bob, index out of range keeps raw label,
  word-level speaker field also updated
- `export_speakers_audio`: no ffmpeg → graceful message; no speaker segments → warning;
  ffmpeg command built correctly with `aselect` filter per speaker

### test_rss.py (HTTP mocked)
- Valid feed with audio enclosures → correct (url, title) tuples, newest-first
- `latest=5` slices correctly; `latest=0` returns all
- Feed with no `<enclosure>` elements → sys.exit(3)
- Network error → sys.exit(3)
- Malformed XML → sys.exit(3)

### test_router.py (bash router via subprocess + stub backends)
- `--check` → exits 2 when no backends installed; JSON output has `ready`, `action`, `backends` fields
- `--check --json` → valid JSON
- `--probe audio.wav` → calls Python probe, returns JSON per file
- `--version` → prints `super-transcribe X.Y.Z`
- `--backends` → lists both backends
- Routing: `--translate` → faster-whisper stub called; `--fast` → parakeet stub called;
  `--long-form` → parakeet; `--streaming` → parakeet; non-EU `-l ja` → faster-whisper;
  default (no flags) → parakeet (if available)
- Conflicting flags (`--translate --long-form`) → exit 3
- `--backend fw` / `--backend pk` aliases → normalized correctly
- `--backend bogus` → exit 3
- `--setup` without value → exit 1

## Integration Test Coverage

### Fixture
`tests/fixtures/tone.wav` — 1-second 440Hz sine wave at 16kHz mono PCM, generated by
`conftest.py` using `numpy` + stdlib `wave`. Committed to repo. Also used in unit tests
for audio helper tests.

### Skip Strategy
Session-scoped fixtures check for backend venv presence:
- faster-whisper: `scripts/backends/faster-whisper/.venv/bin/python`
- parakeet: `scripts/backends/parakeet/venv/bin/python`

If absent, all tests in that file skip with a descriptive message.

### test_fw_e2e.py
- Basic transcription of `tone.wav` → exit 0, stdout non-empty or empty (silence OK)
- `--format srt` → valid SRT (cue numbers, `-->` arrows, blank-line separators)
- `--format json` → parseable JSON with `text`, `language`, `segments`, `duration` keys
- `--agent` → single-line JSON to stdout; all agent fields present (`text`, `duration`,
  `language`, `backend`, `segments`, `word_count`)
- `--probe tone.wav` → JSON with `duration` ~1.0, `channels`, `sample_rate`, `format`
- `--detect-language-only` → prints `Language: XX (probability: ...)` line, exits 0
- `--skip-existing` → second run on same file skips without re-transcribing
- Non-existent input file → exits 3
- `--format invalid_fmt` → exits with argparse error (non-zero)

### test_pk_e2e.py
- Basic transcription of `tone.wav` → exit 0
- `--format srt` → valid SRT structure
- `--format json` → parseable JSON with required fields
- `--agent` → single-line JSON with all agent fields
- `--probe tone.wav` → JSON with duration/channels/sample_rate
- `--fast` flag → exits 0 (uses 110M model alias)
- `--no-align` → exits 0 (skips wav2vec2 alignment path)
- Non-existent input file → exits 3
